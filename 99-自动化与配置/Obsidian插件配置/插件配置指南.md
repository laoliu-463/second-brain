# Obsidian插件配置指南

> 本指南用于配置第二大脑的核心插件：Templater、Linter、Git

---

## 一、Templater 配置

### 1.1 基础设置

打开 Obsidian 设置 → 社区插件 → Templater → 设置：

| 配置项 | 值 |
|--------|-----|
| 模板文件夹 | `09-模板` |
| 触发快捷键 | 启用 |

### 1.2 模板快捷键映射

| 快捷键 | 模板文件 |
|--------|----------|
| `Ctrl+Shift+I` | 收件箱模板-精简版.md |
| `Ctrl+Shift+K` | tpl-核心知识点.md |
| `Ctrl+Shift+Q` | tpl-面试题拆解.md |
| `Ctrl+Shift+L` | tpl-课程学习.md |
| `Ctrl+Shift+P` | 项目模板.md |
| `Ctrl+Shift+R` | 阅读模板.md |

### 1.3 自动元数据增强

在每个模板的 YAML frontmatter 中添加：

```yaml
create_time: <% tp.date.now("YYYY-MM-DD HH:mm") %>
update_time: <% tp.date.now("YYYY-MM-DD HH:mm") %>
```

## 二、Linter 配置

### 2.1 必开规则

打开 Obsidian 设置 → 社区插件 → Linter → 规则配置：

#### 标题规则
- [x] 强制标题层级从H1开始
- [x] 修正标题层级顺序

#### 代码块规则
- [x] 代码块语言标识补全
- [x] 统一代码块围栏为 ```

#### 链接规则
- [x] 统一内部链接格式
- [x] 统一外部链接格式

#### 空格规则
- [x] 去除行尾空格
- [x] 标题前后添加空行

### 2.2 快捷键

- **格式化快捷键**: `Ctrl+Alt+L`

### 2.3 YAML配置（可选）

如需导入配置，可使用以下JSON：

```json
{
  "headingStyle": {
    "style": "atx"
  },
  "blockquoteStyle": {
    "style": "space"
  },
  "listStyle": {
    "type": "bullet"
  },
  "removeLeadingSpaces": true,
  "removeTrailingSpaces": true,
  "codeFence": "fence",
  "languageMappings": {}
}
```

## 三、Git 插件配置

### 3.1 自动同步设置

打开 Obsidian 设置 → 社区插件 → Obsidian Git → 设置：

| 配置项 | 值 |
|--------|-----|
| 自动提交 | 启用 |
| 提交间隔 | 30 分钟 |
| 自动推送 | 启用（提交后立即推送） |
| 启动时自动拉取 | 启用 |

### 3.2 提交信息模板

```
{{type}}: {{message}}
```

### 3.3 仓库设置

确保 `.gitignore` 已配置以下内容：

```
# Obsidian 缓存
.obsidian/
.trash/
.DS_Store

# 大文件
*.pdf
*.zip
*.mp4
*.png
*.jpg

# 临时文件
99-附件/
99-自动化与配置/temp/
99-自动化与配置/AI输出/
```

## 四、快速验证

### 4.1 测试 Templater

1. 按 `Ctrl+N` 新建笔记
2. 按 `Ctrl+Shift+K`
3. 确认模板正确插入

### 4.2 测试 Linter

1. 创建包含格式问题的测试笔记
2. 按 `Ctrl+Alt+L`
3. 确认格式已修正

### 4.3 测试 Git

1. 修改任意笔记
2. 等待30分钟或手动触发提交
3. 确认GitHub已更新
