# 第四阶段（第 4 部分）：测试样例集设计

---

## 一、先拆解需求

这份 `测试样例集` 至少要覆盖 6 个验证维度：

1. **分类准确性** - AI 能否正确判断笔记类型
2. **目录归位** - AI 能否把笔记放到正确目录
3. **Frontmatter 补全** - 必填字段是否完整
4. **双向链接** - 该链接的能不能连上，不该连的能不能刹住车
5. **参考资料** - 能不能正确处理真实来源 vs 猜测来源
6. **审核输出** - 格式是否稳定，是否满足最低合格标准

每维度需要 **3-5 个代表性样例**，覆盖常规场景和边界场景。

---

## 二、分析：测试样例怎么设计

### 1. 样例设计的核心原则

#### A. 覆盖主要笔记类型

| 类型 | 需要几个样例 | 样例特征 |
|------|-------------|----------|
| project（项目） | 2-3 | 明确项目名 / 无项目名 / 项目+知识混杂 |
| reading（阅读） | 2-3 | 有来源链接 / 无来源链接 / 多来源 |
| idea（灵感） | 2 | 纯灵感 / 灵感可转知识 |
| daily（日常） | 1-2 | 常规日志 / 待办为主 |
| knowledge（永久笔记候选） | 2 | 成熟内容 / 草稿内容 |
| 暂不确定 | 1-2 | 用户勾选"暂不确定" |

#### B. 覆盖边界场景

- 用户分类和正文语义冲突
- 有关联项目 vs 无关联项目
- 有来源链接 vs 无来源链接
- 单条来源 vs 多条来源
- 内容混杂（需要拆分评估）
- 低风险 vs 高风险混在一起

#### C. 样例要有可验证性

每个样例必须自带**预期结果**，便于后续对比 AI 输出与预期。

### 2. 样例结构

每个测试样例应该包含：

```markdown
## 样例编号：XX
## 类型：project/reading/idea/daily/knowledge/暂不确定

### 收件箱模板输入
- 标题：
- 内容类型勾选：
- 关联信息：
- 来源信息：
- AI整理要求：
- 特别说明：

### 正文内容
（实际笔记内容）

### 预期结果
- 主类型预期：
- 目录预期：
- Frontmatter 预期字段：
- 预期已确认链接：
- 预期候选链接：
- 预期参考资料：
- 风险等级：
- 预期审核输出类型：完整版/精简版
```

### 样例 01：项目推进 - 明确项目名

#### 收件箱模板输入
- **标题**：`第二大脑系统目录结构讨论`
- **内容类型勾选**：`项目推进`
- **关联项目**：`[[项目-第二大脑搭建]]`
- **来源信息**：无
- **AI整理要求**：`优化格式、补充metadata、推荐双向链接、归类到正确目录`
- **特别说明**：`不要改动我对目录结构的原始想法`

#### 正文内容
```
今天和 AI 讨论了第二大脑的目录结构设计。

初步结论是用 00-09 前缀分层：
- 00-Inbox
- 10-Daily
- 20-Projects
- 30-Areas
- 40-Resources
- 50-Knowledge
- 60-MOCs
- 70-Reviews
- 80-Archive
- 90-System

AI 建议用这种数字前缀，优点是排序稳定。我同意这个方向，但细节还需要再定。
```

#### 预期结果
- **主类型预期**：`project`
- **目录预期**：`03-项目/进行中`
- **Frontmatter 预期**：`type: project, project: 项目-第二大脑搭建, status: active`
- **预期已确认链接**：`[[项目-第二大脑搭建]]`
- **预期候选链接**：`[[MOC-知识管理]]`
- **预期参考资料**：无外部链接
- **风险等级**：低
- **预期审核输出类型**：完整版

## 四、测试样例使用方式

### 1. 人工对照测试

每个样例由 AI 处理，对照预期结果逐项检查：

- 分类是否正确
- 目录是否正确
- Frontmatter 是否完整
- 链接是否合理
- 参考资料是否合规
- 审核输出是否满足最低标准

### 2. 批量回归测试

后续规则有变更时，用这批样例重新跑一遍，确保不引入回归。

### 3. 准确率统计

| 维度 | 合格标准 |
|------|---------|
| 分类准确率 | >= 80% |
| 目录归位准确率 | >= 85% |
| Frontmatter 完整率 | >= 90% |
| 链接准确率 | >= 80% |
| 参考资料合规率 | 100%（不允许伪造） |
