# 整理检查清单

本清单用于检查 AI 对收件箱笔记的整理结果是否合格。

适用场景：
- 人工审核 AI 整理结果
- AI 在输出整理结果前进行自检
- 审核模式下判断哪些动作可以执行
- 判断一篇笔记是否允许进入正式知识库

---

## 1. 基础信息检查

- [ ] 已明确本次整理对象是哪篇收件箱笔记
- [ ] 已明确整理模式（保守模式 / 审核模式 / 自动模式）
- [ ] 已输出本次分类结果
- [ ] 已输出目标目录或保留收件箱的决定
- [ ] 已说明是否新建正式笔记或更新已有笔记
- [ ] 已说明本次整理依据了哪些用户输入信息

若以上任一项缺失，则本次整理结果不完整。

---

## 2. 分类检查

- [ ] 分类结果优先依据用户填写的“内容类型”
- [ ] 若用户填写了关联信息，分类时已参考关联信息
- [ ] 若用户填写了来源信息，分类时已参考来源信息
- [ ] 没有因为自由语义猜测而推翻用户明确填写的内容类型
- [ ] 多标签冲突时已进行保守处理
- [ ] 若分类不明确，已降级为候选类型或保留在收件箱
- [ ] 没有把“灵感想法”直接强行定稿为永久笔记
- [ ] 没有把“暂不确定”强行归入正式类型

若分类判断与用户标签明显冲突，则本次整理结果不合格。

---

## 3. 目录归类检查

- [ ] 已先判断是否应离开收件箱，而不是直接移动
- [ ] 目标目录与正式类型一致
- [ ] 若目录不明确，已选择保留在收件箱
- [ ] 若更新已有笔记，目标笔记真实存在
- [ ] 没有重复新建与已有笔记高度相似的正式笔记
- [ ] 没有在低置信度情况下自动移动到正式目录
- [ ] 若执行了目录移动，该动作符合当前执行模式或已审核确认
- [ ] 原始收件箱笔记已保留

若目录归类存在疑问但仍被强行执行，则本次整理结果不合格。

---

## 4. frontmatter 检查

- [ ] 已补全最小必要字段
- [ ] `type` 字段存在且正确
- [ ] `title` 字段存在且与笔记标题基本一致
- [ ] `created` 字段存在
- [ ] `updated` 字段存在
- [ ] 若来自收件箱，`source_note` 字段已补充
- [ ] 若为 `daily`，已补 `date`
- [ ] 若为 `project`，已补合理的 `status`
- [ ] 若为 `reading`，在来源明确时已补 `source`
- [ ] 没有为了追求完整性而虚构字段值
- [ ] 没有在无依据时自动填写高风险字段（如 `author`、`aliases`、`priority`、`review_cycle`、`area`、`progress` 等）

若 frontmatter 中存在明显猜测性字段，则本次整理结果有风险。

---

## 5. 正文格式检查

- [ ] 标题已优化为清晰可读形式，且未改变原意
- [ ] 正文已完成基础段落整理
- [ ] 列表、任务、引用格式已规范化
- [ ] 未擅自扩写用户未表达的内容
- [ ] 未把探索性表达改写为确定性结论
- [ ] 未擅自补充未经验证的事实
- [ ] 若涉及来源内容，已区分“用户理解”与“外部摘录”
- [ ] 已按需要增加“来源笔记”区
- [ ] 已按需要增加“相关笔记”区
- [ ] 已按规则处理“参考资料”区

若正文可读性提升但含义发生变化，则本次整理结果不合格。

---

## 6. 内部链接检查

- [ ] 自动建链优先依据用户填写的关联信息
- [ ] 所有自动插入的目标笔记都真实存在
- [ ] 没有因为词面相似而强行建链
- [ ] 没有生成悬空内部链接
- [ ] 若关系不明确，已输出“候选关联”而非自动插入正文链接
- [ ] 正文中的自动链接数量克制，没有造成阅读噪音
- [ ] 若用户未允许自动插入内部链接，则未修改正文链接
- [ ] 若仅适合文末整理，已优先使用“相关笔记”区而非正文插链

若存在误链、悬空链接或不确定关系的强插链接，则本次整理结果不合格。

---

## 7. 参考资料检查

- [ ] 若正文涉及外部资料，已检查是否存在具体来源
- [ ] 每条参考资料都包含“来源名称 + 具体链接”
- [ ] 没有只写模糊来源名而缺少具体链接
- [ ] 没有伪造参考文章链接
- [ ] 没有根据猜测补全来源页面
- [ ] 若原始输入未提供具体链接，已明确标记“待补充”
- [ ] 内部来源与外部参考已分开呈现
- [ ] 若来源不足，已降级为草稿整理而非正式参考型定稿

若参考资料区存在伪造、猜测或不可验证链接，则本次整理结果直接不合格。

---

## 8. 权限与执行边界检查

- [ ] 本次执行动作符合当前执行模式
- [ ] 低风险动作可自动执行的部分未越界
- [ ] 中高风险动作已正确标记为“需确认后执行”
- [ ] 未自动删除原始收件箱笔记
- [ ] 未自动改写永久笔记核心正文
- [ ] 未自动合并多个正式笔记
- [ ] 未在无确认情况下执行高风险目录移动
- [ ] 未在无确认情况下执行高风险正文插链
- [ ] 未在无确认情况下拆分一篇笔记为多篇

若存在越权执行，则本次整理结果不合格。

---

## 9. 降级处理检查

- [ ] 若分类不明确，已保留在收件箱或输出候选类型
- [ ] 若目录不明确，未强行移动
- [ ] 若链接不明确，未强行插链
- [ ] 若来源不明确，未强行补参考资料
- [ ] 若内容混杂，已标记“建议拆分”
- [ ] 若整理风险较高，已降级为草稿整理或审核模式输出

若本应降级却被强行正式化，则本次整理结果不合格。

---

## 10. 最终结果判定

### A. 合格，可进入正式库
满足以下条件时可判为合格：
- 分类合理
- 目录合理
- frontmatter 合格
- 无明显误链
- 无伪造来源
- 原始笔记保留
- 未越权执行

### B. 有风险，可保留为审核结果
满足以下情况时应进入审核而非直接执行：
- 分类基本明确但仍有冲突
- 目录大体合理但更新目标需确认
- 链接候选存在但不够明确
- 来源部分缺失但未伪造
- 涉及高风险动作尚未确认

### C. 不合格，应退回收件箱或草稿状态
存在以下任一情况时，应判不合格：
- 分类明显错误
- 目录明显错误
- 误建链接
- 伪造参考资料
- frontmatter 乱填
- 越权执行
- 擅自改写原意
- 原始收件箱笔记被删除或不可追溯

---

---

## 12. 最低合格线

一次整理结果至少要满足以下条件，才可视为最低合格：

- [ ] 没有伪造来源
- [ ] 没有误链
- [ ] 没有明显错分类
- [ ] 没有明显错目录
- [ ] frontmatter 最小字段完整
- [ ] 原始收件箱笔记保留
- [ ] 没有越权执行高风险动作

若以上任一项不满足，则不应直接进入正式知识库。
