# AI整理规则 v1（审核模式默认）

## 0. 文档目的

本文件用于约束 AI 对 `01-收件箱/` 笔记的整理行为，确保：

1. 目录归类正确
2. 链接建立正确
3. frontmatter 稳定可用
4. 参考资料可验证且不伪造
5. 原始内容可追溯、可回滚

---

## 1. 适用范围

- 输入来源：`01-收件箱/`
- 输出目标：正式目录体系（按本文件目录映射）
- 默认执行模式：`review`（审核模式）

---

## 2. 核心原则

1. 用户信号优先：`内容类型 > 正文语义 > 关联信息 > 来源信息 > AI整理要求`
2. 宁可少改，不乱改；宁可少链，不误链
3. 无证据不补事实，无链接不造来源
4. 高风险动作先审核，低风险动作可自动
5. 不确定时降级保留在收件箱

---

## 3. 执行模式

### 3.1 mode=conservative（保守）

- 自动：格式整理、最小 frontmatter、来源笔记区
- 不执行：移动目录、改名、正文插链、拆分、更新正式笔记、更新 MOC

### 3.2 mode=review（审核，默认）

- 自动：格式整理、最小 frontmatter、来源笔记区、真实参考链接写入
- 审核后执行：移动目录、改名、正文插链、更新正式笔记、拆分、更新 MOC

### 3.3 mode=auto（自动）

- 自动执行高置信低歧义动作
- 仍禁止：删除原收件箱、伪造来源、改写永久笔记核心正文、合并正式笔记

---

## 4. 权限开关（建议配置）

```yaml
mode: review
allow_move: false
allow_insert_body_links: false
allow_update_existing_notes: false
allow_split_notes: false
allow_update_moc: false
allow_auto_promote_permanent: false
allow_archive: false
```

说明：当开关为 `false` 时，AI 仅输出建议与候选，不执行对应动作。

---

## 5. 目录映射（类型 -> 目录）

- `inbox` -> `01-收件箱/`
- `daily` -> `02-日记/YYYY/YYYY-MM/`
- `project` -> `03-项目/进行中/`（或 `已暂停/已完成`）
- `area` -> `04-领域/`
- `resource` -> `05-资源/`
- `reading` -> `05-资源/阅读/`
- `permanent` -> `06-永久笔记/`
- `moc` -> `07-索引/MOC/`
- `archive` -> `08-归档/`

规则：先判断“是否转正”，再判断“放哪里”。不确定则保留 `01-收件箱/`。

---

## 6. 分类与转正规则（摘要）

1. 优先读取收件箱模板中的 `内容类型（可多选）`
2. 正文语义用于校验，不应轻易推翻用户标签
3. `暂不确定` 为强降级信号：默认不转正
4. 灵感默认保留 inbox，标记为 `permanent_candidate`（候选）
5. 仅在内容成熟、来源清晰、关系明确时才转 `permanent`

---

## 7. frontmatter 自动补全规则

### 7.1 通用最小字段

所有正式笔记优先补：

- `type`（必填）
- `title`（建议必填）
- `created`（必填）
- `updated`（建议必填）
- `source_note`（由收件箱转正时强烈建议必填）
- `tags`（可选，1~5 个高价值标签）

### 7.2 类型专属最小字段

- `daily`: `date`
- `project`: `status`（默认 `active`，仅在无冲突时）
- `reading`: `source`（来源名称明确时）
- `moc`: `topic`
- `archive`: `archived`

### 7.3 禁止瞎填字段

无充分依据时禁止自动填：

- `author`
- `aliases`
- `priority`
- `review_cycle`
- `owner`
- `area`
- `end_date`
- `progress`

---

## 8. 格式优化规则

允许：

- 标题轻优化、分段、小标题整理、列表/任务语法规范、引用块规范

禁止：

- 擅自改写观点
- 擅自补事实
- 擅自给出确定性结论

正文建议结构：

1. `## 概要`
2. `## 正文`
3. `## 相关笔记`
4. `## 来源笔记`
5. `## 参考资料`

---

## 9. 参考资料与低幻觉规则（强制）

1. 涉及外部内容时，`参考资料` 每条必须包含“来源名称 + 具体 URL”
2. 无具体链接时必须显式写“待补充”，严禁伪造链接
3. 内部来源与外部来源分开写
4. 事实补充必须有证据锚点（用户文本、已给链接、系统已知来源）

降级模板：

```markdown
## 参考资料
- 待补充：原始输入未提供可验证外部链接
```

---

## 10. 双向链接生成规则

自动建链前必须同时满足：

1. 目标笔记真实存在
2. 关系明确（用户指定或正文明确指向）
3. 无同名/近义冲突
4. 权限允许（例如允许自动插正文链接）

优先级：

1. 保留用户已有链接
2. 优先写入文末 `相关笔记`
3. 仅在必要时插入正文

不确定则输出 `候选关联`，不强插。

---

## 11. 动作分级

### L1 自动执行（低风险）

- 格式整理
- 最小 frontmatter
- 来源笔记区
- 真实参考链接写入
- 高置信文末相关笔记补链

### L2 审核后执行（中高风险）

- 目录移动
- 文件重命名
- 正文插链
- 更新已有正式笔记
- 拆分笔记
- 更新 MOC/索引

### L3 只建议（禁止自动）

- 删除收件箱原文
- 改写永久笔记核心正文
- 合并多个正式笔记
- 删除既有链接
- 伪造外部参考链接

---

## 12. 整理流水线

1. 读取笔记与模板区块
2. 生成分类结果（主类型/次类型/置信度）
3. 判断是否转正（立即转正/候选转正/保留收件箱）
4. 生成 frontmatter（先最小字段）
5. 格式整理（不改事实）
6. 处理链接（已确认 + 候选分离）
7. 处理参考资料（真实链接或待补充）
8. 输出执行计划（区分自动/审核/仅建议）
9. 执行允许动作并记录审计信息

---

## 13. 审核模式输出格式（固定）

```markdown
整理模式：review

分类结果：<type>
目标目录：<path>
处理动作：<新建|更新|保留收件箱>
目标笔记：<[[note]] 或 N/A>

可自动执行：
- ...

需确认后执行：
- ...

仅给出建议：
- ...

参考资料状态：
- 已确认链接：N 条
- 待补充：M 条

风险说明：
- ...
```

---

## 14. 正确性检查清单（执行后）

- [ ] 主类型与用户内容类型不冲突
- [ ] 目录落位符合映射规则
- [ ] 未重复新建同义正式笔记
- [ ] frontmatter 最小字段完整
- [ ] 内部链接目标真实存在
- [ ] 无悬空/猜测性链接
- [ ] 外部参考含具体 URL 或明确待补充
- [ ] 无伪造来源
- [ ] 原收件箱笔记已保留并可追溯
- [ ] 未越权执行高风险动作

---

## 15. 失败与降级策略

- 分类不明确 -> 保留收件箱 + 候选类型
- 目录不明确 -> 不移动目录 + 仅格式整理
- 链接不明确 -> 候选关联，不插正文
- 来源不明确 -> 参考资料写待补充，不转正式参考型定稿
- 内容混杂 -> 建议拆分，不自动拆分

---

## 16. Git 提交建议

1. `note(inbox):` 原始输入
2. `chore(ai-draft):` AI 自动低风险整理
3. `chore(ai-reviewed):` 审核通过后的结构性变更（移动/改名/更新索引）

要求：结构性动作单独提交，便于回滚与审计。

---

## 17. MVP 验收门槛

1. AI 能稳定读取收件箱模板字段
2. 样例集主分类正确率 >= 80%
3. 不伪造参考链接
4. 不确定场景可降级而非强行整理
5. 审核输出结构稳定可复核
6. 原始收件箱内容可追溯
