# 执行手册（审核模式）

## 1. 目标

把你现有流程跑通：

1. 收件箱写作
2. AI 审核输出
3. CLI 执行低风险动作
4. 人工确认高风险动作
5. Git 三段提交

## 2. 脚本清单

- `00-系统/scripts/obsidian-review-flow.ps1`
- `00-系统/scripts/git-review-flow.ps1`

## 3. 最小执行链路

### Step A：检查环境

```powershell
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/obsidian-review-flow.ps1" -Action status
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/git-review-flow.ps1" -Action status
```

### Step B：AI 输出审核结果（review）

确保输出包含：
- 主类型
- 目标目录
- 已自动执行
- 需确认后执行
- 链接状态
- 参考资料状态
- 风险说明

### Step C：执行低风险动作

```powershell
# 搜索目标笔记
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/obsidian-review-flow.ps1" -Action search -Query "项目-第二大脑搭建"

# 创建正式草稿
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/obsidian-review-flow.ps1" -Action create_note -Path "03-项目/进行中/项目-第二大脑搭建.md" -Content "# 项目-第二大脑搭建"

# 追加今日日记审核记录
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/obsidian-review-flow.ps1" -Action daily_append -Content "- [ ] 审核：项目-第二大脑搭建（待确认）"
```

### Step D：三段提交

```powershell
# 1) 原始输入提交（默认收件箱目录）
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/git-review-flow.ps1" -Action commit_inbox -Message "add raw note"

# 2) AI 草稿提交（明确路径）
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/git-review-flow.ps1" -Action commit_ai_draft -Paths "03-项目/进行中/项目-第二大脑搭建.md","02-日记/2026/2026-03/2026-03-18.md" -Message "generate structured draft"

# 3) 审核通过后提交（明确路径）
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/git-review-flow.ps1" -Action commit_ai_reviewed -Paths "03-项目/进行中/项目-第二大脑搭建.md","07-索引/MOC/MOC-知识管理.md" -Message "apply approved actions"
```

## 4. 安全边界

- `commit_ai_draft` / `commit_ai_reviewed` 强制要求 `-Paths`，防止误提交。
- 高风险动作（移动目录、正文插链、更新既有正式笔记）不自动执行。
- 无外部真实链接时，参考资料必须写“待补充”。

## 5. 常用预演（不落地）

```powershell
powershell -ExecutionPolicy Bypass -File "00-系统/scripts/git-review-flow.ps1" -Action commit_ai_draft -Paths "03-项目/进行中/项目-第二大脑搭建.md" -Message "test" -DryRun
```
