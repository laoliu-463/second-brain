# 第三阶段（第 1 部分-A）：收件箱模板正式定稿

## 1. 目标与结论

本轮定稿采用「轻量半结构化模板」，并使用双模板策略：

1. `tpl-inbox.md` 作为默认精简版（优先速度）  
2. `tpl-inbox-formal.md` 作为正式版（优先准确率）  
3. 两版都支持“先写正文，再补结构化信号”

## 2. 定稿原则

1. 正文优先：`原始内容` 永远放第一位  
2. 低负担：大多数字段可留空  
3. 高可解析：仅保留高价值信号（类型、关联、来源、权限边界）  
4. 中文友好：提示语与区块名均为中文  
5. 权限内建：模板直接控制 AI 的执行边界

## 3. 模块定稿（6 区块）

1. 标题  
2. 原始内容  
3. 内容类型（可多选）  
4. 关联信息（项目/领域/主题/笔记）  
5. 来源信息（按需填写）  
6. AI 整理要求 + 特别说明

## 4. 正式版模板（tpl-inbox-formal.md）

```markdown
# {{标题}}

## 原始内容
直接记录你的内容，可以是想法、项目进展、阅读摘录、问题、草稿或待办。

## 内容类型（可多选）
- [ ] 项目推进
- [ ] 日常记录
- [ ] 阅读笔记
- [ ] 资料整理
- [ ] 灵感想法
- [ ] 长期知识候选
- [ ] 待办事项
- [ ] 暂不确定

## 关联信息（知道就填，不知道可留空）
- 关联项目：
- 关联领域：
- 关联主题：
- 关联笔记：

## 来源信息（如果有）
- 来源名称：
- 来源链接：

## AI 整理要求（可多选）
- [ ] 优化标题
- [ ] 优化格式
- [ ] 补充 metadata
- [ ] 推荐标签
- [ ] 推荐双向链接
- [ ] 归类到正确目录
- [ ] 更新相关索引
- [ ] 必要时拆分内容
- [ ] 允许自动移动到正式目录
- [ ] 允许自动插入内部链接
- [ ] 保留原文，仅轻度整理

## 特别说明
- 不要改动的内容：
- 我最关心的重点：
- 其他备注：
```

## 5. 默认精简版模板（tpl-inbox.md）

```markdown
# {{标题}}

## 原始内容
直接写内容。

## 内容类型（可多选）
- [ ] 项目推进
- [ ] 日常记录
- [ ] 阅读笔记
- [ ] 灵感想法
- [ ] 长期知识候选
- [ ] 暂不确定

## 关联信息
- 关联项目：
- 关联主题：
- 关联笔记：

## AI 整理要求
- [ ] 优化格式
- [ ] 补充 metadata
- [ ] 推荐双向链接
- [ ] 归类到正确目录
- [ ] 保留原文，仅轻度整理
```

## 6. 使用策略（执行版）

1. 日常默认用精简版：提高输入频率  
2. 阅读长文、重整理场景用正式版：提高分类与建链准确率  
3. 勾选 `允许自动移动到正式目录` 才进入“自动执行模式”  
4. 勾选 `保留原文，仅轻度整理` 时，AI 只做轻改不重写

## 7. 本部分验收标准

1. 双模板文件都已落地于 `90-System/templates/`  
2. 默认模板为精简版，可一键快速输入  
3. 正式版包含自动移动/自动建链权限位与特别说明区  
4. 后续“第 2 部分分类判定规则”可直接消费两版字段
